## Multi-Model Analysis

Three models were run in parallel (background agents) to independently investigate the root cause and propose a fix.

### ­čĄľ Model 1 ÔÇö Claude Sonnet 4.6 (This session ÔÇö implemented fix)

**Root Cause:**
`OnHandlerChanged` with `Handler == null` sets `HintLabelStyle`, `HelperLabelStyle`, and `ErrorLabelStyle` to `null`. On reconnect, they remain `null`. `DrawHintText` has a `HintLabelStyle != null` guard ÔÇö so hint is never drawn.

**Fix:** In the `Handler != null` branch, call `ClearValue()` on any null style before `WireEvents()` to trigger the `defaultValueCreator` and reinstate defaults.

```csharp
if (Handler != null)
{
    if (HintLabelStyle == null)   ClearValue(HintLabelStyleProperty);
    if (HelperLabelStyle == null) ClearValue(HelperLabelStyleProperty);
    if (ErrorLabelStyle == null)  ClearValue(ErrorLabelStyleProperty);
    WireEvents();
    OnTextInputViewHandlerChanged(...);
}
else { /* unchanged */ }
```

**Risk:** Low. Keeps the `else` block intact; only adds recovery on reconnect.
